¿cómo implemento un websocket en rust usando tokio?

Kali

Para un WebSocket en Rust con Tokio te recomiendo tokio-tungstenite. Es la capa estándar sobre tungstenite, integrada con el runtime asíncrono. Aquí va un ejemplo mínimo de servidor echo: 

// Cargo.toml: tokio-tungstenite = "0.20"
use tokio_tungstenite::accept_async;
use futures_util::StreamExt;

async fn handle(stream: TcpStream) {
    let mut ws = accept_async(stream).await.unwrap();
    while let Some(msg) = ws.next().await {
        if let Ok(m) = msg {
            ws.send(m).await.unwrap();
        }
    }
}

¿y cómo manejo errores sin que se caiga todo el servidor?

Kali · generando

El patrón clave es no propagar el error del handler individual al accept loop. Cada conexión se aisla con un spawn que loguea y termina sin matar el listener. Además conviene un timeout por conexión para evitar que una peer muerta te ocupe un slot indefinidamente. Te lo muestro en dos
